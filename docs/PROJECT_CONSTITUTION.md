# RelayNet 项目宪法（Project Constitution）

**版本**：v1.0  
**状态**：生效  
**适用范围**：RelayNet 全部仓库与全部贡献形式  
**法律地位**：高于设计文档、高于实现代码、高于临时决策  

---

## 1. 宪法地位声明（最高约束）

本文件是 **RelayNet 项目的最高约束性文件**。

- 所有架构设计、协议定义、代码实现、讨论结论，均不得违反本宪法
- 当本宪法与 README、设计文档、Issue、PR 描述发生冲突时：
  - **以本宪法为最终裁决依据**
- 本宪法对以下对象具有同等约束力：
  - 所有人类贡献者
  - 所有 AI 辅助系统（包括但不限于 ChatGPT）

---

## 2. 项目本质与不可动摇定位

RelayNet 是一个：

> **跨组织物流责任接力的规则型平台与协议基础设施**

RelayNet **不是**：
- TMS / WMS / OMS
- 运力撮合或调度系统
- 业务应用或 SaaS 产品

RelayNet 关注的唯一核心对象是：
- **责任接力（Responsibility Handoff）**
- **状态转移（State Transition）**
- **可审计事件（Auditable Events）**

---

## 3. 仓库分权与不可越界原则（核心）

### 3.1 relaynet-core（规则与协议内核）

`relaynet-core` 是 **规则与协议的唯一真源（Single Source of Truth）**。

**允许包含**：
- 接力任务与阶段模型
- 状态机定义
- 事件与审计模型
- 协议级 API 定义

**明确禁止**：
- UI / 前端代码
- 行业或业务逻辑
- SDK 便利封装
- 企业交付与部署逻辑

---

### 3.2 relaynet-sdk（接入工具层）

`relaynet-sdk` 是 **开发者接入工具层**。

**允许包含**：
- 客户端封装（HTTP / gRPC 等）
- 参数校验
- 开发者友好接口

**明确禁止**：
- 规则判断
- 状态机实现
- 协议定义
- 任何“替 core 做决定”的逻辑

---

### 3.3 relaynet-enterprise（商业边界）

`relaynet-enterprise` 是 **商业与私有化边界仓库**。

**允许包含**：
- 商业部署与运维工具
- 行业模板与企业扩展
- SLA、监控与合规模块

**明确禁止**：
- 回流污染 core 的规则逻辑
- 复制 core / sdk 源码
- 破坏开闭源边界的实现

---

## 4. AI / ChatGPT 辅助编程专用条款

- AI 输出内容一律视为 **辅助性建议**
- AI 不得在未提示风险的情况下：
  - 突破仓库职责边界
  - 模糊开源 / 闭源界限
  - 在 SDK 或 Enterprise 中重建规则逻辑
- 最终设计与采纳责任 **始终由人类维护者承担**

---

## 5. 协议优先与冻结原则

- **协议必须先于代码**
- 核心语义一经冻结，不得因实现便利而修改
- 如需突破冻结内容，必须：
  - 明确提出修宪
  - 说明理由与影响范围

---

## 6. 裁决与修订

- 所有争议以本宪法为最终裁决依据
- 本宪法允许修订，但必须：
  - 明确版本号
  - 明确修订理由
  - 不得追溯破坏既有协议与边界

---

## 7. 生效声明

本宪法自提交之日起立即生效，  
并对 RelayNet 项目内所有仓库与所有贡献形式产生持续约束力。
